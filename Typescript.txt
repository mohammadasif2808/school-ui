 This is a significant architectural change. Converting to TypeScript will improve code quality, autocomplete, and maintainability, but it requires a
  systematic approach to avoid breaking the build.

  Here is the TypeScript Migration Blueprint.

  Blueprint: JavaScript to TypeScript Migration

  Phase 1: Foundation & Tooling
   1. Install Dependencies:
       * Install typescript and type definitions (@types/react, @types/react-dom, @types/node).
       * Install types for existing libraries if needed (@types/recharts usually not needed as it has built-in types, but we will check).
   2. Configuration:
       * Create tsconfig.json (TypeScript configuration).
       * Create src/vite-env.d.ts (Vite type definitions for assets like SVG and CSS Modules).
       * Update vite.config.js to vite.config.ts.

  Phase 2: Core Logic & Types
   3. Define Global Interfaces (`src/types/index.ts`):
       * Create shared interfaces for User, Student, DashboardStats, etc., to avoid repetition.
   4. Convert Services (`.js` -> `.ts`):
       * src/services/api.js → src/services/api.ts (Type Axios instances).
       * src/services/authService.js → src/services/authService.ts (Type return values and payloads).

  Phase 3: State Management (Contexts)
   5. Convert Contexts (`.jsx` -> `.tsx`):
       * ThemeContext.jsx → ThemeContext.tsx (Type the context value: { theme, toggleTheme }).
       * AuthContext.jsx → AuthContext.tsx (Type the user state and login/logout functions).

  Phase 4: Component Migration
  Strategy: Migrate from "Leaf" components (smallest) to "Root" components.

   6. Layout Components:
       * Sidebar.jsx → Sidebar.tsx.
       * Header.jsx → Header.tsx.
       * MainLayout.jsx → MainLayout.tsx.
       * ProtectedRoute.jsx → ProtectedRoute.tsx.
   7. Feature Modules:
       * LoginPage.jsx → LoginPage.tsx.
       * DashboardPage.jsx → DashboardPage.tsx.
   8. Entry Points:
       * App.jsx → App.tsx.
       * main.jsx → main.tsx.

  Phase 5: Cleanup & Verification
   9. Remove PropTypes: Delete prop-types library and imports (TypeScript Interfaces replace them).
   10. Strict Check: Run tsc (TypeScript Compiler) to find and fix any remaining any types or errors.
